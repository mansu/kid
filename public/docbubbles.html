DocBubbles

Doc Bubbles is a non-linear text documentation format. Unlike a linear textual document which starts at the top left and ends at bottom right of the page, docbubbles allows one to write a document as an interactive and explorable graph. Docbubbles can be thought of a more fine grained and interactive version of mindmap implemented in a browser.

A docbubbles document is made up of several bubbles. The box in which you are viewing this text in is one such bubble. In addition to normal HTML text, a docbubbles document has 2 special kinds of text.

  • InfoNode: An infoNode is a fancy tooltip. For example, an infoNode like this text will provide additional information to a document.
  • NodeLink: Like a hyperlink on a webpage, a node link is a link to a new bubble. Click on this link to open a new bubble in the same page.

Docbubbles is better than a normal page of text with various sections because it will help you include discussions that may be related but orthogonal to current discussion page. For example, click on the next node link to understand a very high level technical overview of docbubble. To close a bubble click on the "X" symbol in the title bar of the bubble.

You can include anything in a bubble. For example, you can have open a webpage like yahoo.com or a full blown linux terminal in a new bubble.

As you can see, unlike the html link that would refresh the current page or open a new link in a new tab/window, the new bubble opens below the current bubble in the same page. Opening the bubble in the same page, helps you keep the context of how you arrived here.

Technically, each bubble in this document is a HTML DIV. A docbubbles page is an HTML page containing a collection of divs containing content written in extended markdown syntax. Since each bubble is a HTML DIV, any HTML content can be shown in a doc bubble.

When the HTML page is opened, all nodes/div's except the first div are hidden. When a node link is clicked, the selected div is displayed on the page. Currently, we use a very simple placement algorithm to place the divs on the page.
